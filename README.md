# 🏡 PGSathi - Comprehensive Project Documentation

PGSathi is a modern, feature-rich web application designed to connect Paying Guest (PG) owners with potential tenants. The platform provides dedicated dashboards for owners to list and manage properties, administrators to monitor the platform, and an intuitive search interface for tenants.

---

## 🚀 1. Tech Stack & Technologies

*   **Frontend & Framework:** [Next.js 16](https://nextjs.org/) (App Router), React 19
*   **Styling & UI:** Tailwind CSS v4, Framer Motion (for animations), Lucide React (icons)
*   **Database ORM:** [Prisma](https://www.prisma.io/) (`@prisma/client` v5.22.0)
*   **Database:** PostgreSQL (Hosted on NeonDB)
*   **Authentication:** NextAuth.js (v5 Beta) with Prisma Adapter
*   **Payments & Monetization:** Razorpay Integration (`razorpay` v2.9.6)
*   **Media & Image Hosting:** Cloudinary
*   **State Management:** Zustand, React Query
*   **Forms & Validation:** React Hook Form, Zod

## 🗄️ 3. Database Architecture (Prisma Models)

The PostgreSQL database is carefully structured using Prisma to handle all relationships seamlessly:

1.  **`User`**: Manages all users with roles (`TENANT`, `OWNER`, `ADMIN`). Stores authentication info, avatars, and relations to their listings or subscriptions.
2.  **`City` & `Locality`**: Hierarchical location data for filtering and organizing PG listings.
3.  **`Listing`**: The core model representing a PG/Property. Includes pricing, rules (gender, PG type), capacity, and verification status (`PENDING`, `ACTIVE`, `REJECTED`).
4.  **`ListingPhoto`**: Stores URLs of images uploaded to Cloudinary, linked to specific listings.
5.  **`Amenity` & `ListingAmenity`**: A junction table linking properties to available amenities (WiFi, AC, Laundry, etc.).
6.  **`Plan` & `Subscription`**: Defines the monetization tiers for owners and tracks their active Razorpay subscriptions. Includes `SubStatus` (`ACTIVE`, `PENDING`, `CANCELLED`).
7.  **`Invoice`**: Tracks individual payment records linked to a subscription.
8.  **`Lead`**: Enquiries generated by tenants for specific properties, visible to the owner.
9.  **`Review`**: Ratings and feedback provided by tenants.
10. **`ListingAnalytic`**: Tracks daily views, leads, and WhatsApp clicks for properties.

## 🔌 4. API Endpoints Overview

| Endpoint | Method | Description |
| :--- | :--- | :--- |
| `/api/auth/[...nextauth]` | `GET/POST` | Handles user authentication via NextAuth |
| `/api/listings` | `GET/POST` | Fetch all listings or create a new `PENDING` listing |
| `/api/cities` | `GET` | Retrieve cities and nested localities for dropdowns |
| `/api/payments/create-order`| `POST` | Initializes a Razorpay order and saves a `PENDING` Subscription |
| `/api/payments/verify` | `POST` | Verifies Razorpay signature and activates the Subscription |
| `/api/upload` | `POST` | Uploads media to Cloudinary and returns secure URLs |

## 📋 5. Application Modules & Features

### 👤 A. Authentication & User Module
*   Supports multiple roles (Admin, Tenant, Owner).
*   Handles secure login and session verification across the platform.

### 🏢 B. Owner Module (Property Management)
*   **Add Listings:** Owners can fill out a multi-step form to list their PG (Location, Amenities, Rules, Pricing, Gallery).
*   **Dashboard:** View the status of their properties, track analytics (views/leads), and manage tenant enquiries.

### 👑 C. Admin Module
*   **Moderation:** Admins have a dedicated dashboard to review all newly submitted (`PENDING`) listings and approve or reject them to maintain quality control.
*   **Platform Stats:** View overall user growth and active subscriptions.

### 💳 D. Payments & Monetization Module
*   Integrated closely with **Razorpay**.
*   Owners can purchase subscription plans or boost their listings for better visibility.
*   Automated invoice generation and database status updates upon successful verification.

### 🔍 E. Tenant Experience (Search & Discovery)
*   Tenants can filter properties by city, rent range, gender rules (Boys/Girls/Co-ed), and specific amenities.
*   Dedicated PG detail pages showcasing gallery, rules, and map locations.

## 💻 7. Local Development Setup

To run this project locally:

1.  Clone the repository.
2.  Install dependencies:
    ```bash
    npm install
    ```
3.  Set up your `.env` file with Database, NextAuth, Cloudinary, and Razorpay credentials.
4.  Push the Prisma schema to your database:
    ```bash
    npx prisma db push
    ```
5.  Start the development server:
    ```bash
    npm run dev
    ```
